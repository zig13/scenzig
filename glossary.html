scenzig Code Glossary

PC = Player Character
NPC = Non-player character

Scene

  • The building blocks of an Adventure

  • A scene is primarily a block of text known as the description. This text is used to present a situation to the player

  • Scenes are used to present a situation to the player for them to resolve using Actions

  • Can represent a physical location or just a conversation.

  • Each scene whitelists and blacklists Actions

Scene State

  • Primarily used to show the PC's influence on a location

  • For example if there is an item in a room and the PC picks it up then the scene state should change to one where the description reflects the item no longer being there.

  • A Scene State can whitelist and blacklist actions and have a different description.

  • Can also be used to keep track of NPC conversations

Action

  • An option available to the player

  • Have effects that change values in the character file

  • Some actions will be generic but many will only be on the whitelist of one specific scene or item.

  • Encompasses activities such as walking through an archway, drinking a potion, putting on a waistcoat, casting a spell, swinging a sword or insulting an NPC.

Effects

  • Change to character condition either temporarily or permanently

  • A temporary effect may come from having an Item present in the inventory or possessing an ability

    • This would included being granted the combat action "Stab" while having a dagger equipped

    • Temporary effects to not directly edit the character file

    • The effects of items and abilities are applied when the character is loaded

  • A permanent effect is the direct result of an Action

    • This would include changing the scene or adding an item to the PC inventory

    • Permanent effects directly alter the character file

Item

  • Represent physical objects in the player's inventory.

  • The Inventory command will list all items the player is carrying

  • Items can whitelist Actions and alter variables

  • A sword could whitelist the action 'Equip sword' which would remove the sword from your inventory and replace it with the item 'Sword (equipped)' which grants the action 'Stab with sword'.

  • Items will also be able to alter character attributes

Ability

  • An alternative way of imparting effects

  • Work very similarly to Items but are invisible to the player

  • Makes it easier to add/remove a group of effects on the PC

  • An ability called super-strength could boost strength by 2 and whitelist the Action 'flex biceps'.


Action Group


  • Allows multiple Actions to be whitelisted or blacklisted together

  • A common usage is to blacklist combat Actions on encounter 0


Encounter


  • Overlays on top of a Scene

  • Can whitelist and blacklist actions

  • Is returned to 0 (no encounter) by meeting a condition 


Attributes

  • Arbitrary values

  • Unlike currencies, the actual numerical attribute value is hidden from the player and instead they are presented with qualitative descriptions of the values

  • Most commonly used to represent prowess or lack thereof of the PC.

  • In the future will be able to be used as a requirement for Actions


Vitals

  • Arbitrary values

  • Like attributes, vitals will be able to used as a requirement

  • Like Attributes, their numerical values are hidden

  • If any PC Vital reaches 0, then the PC fails/dies/loses. The character file is deleted and the game exited.?

  • Success in a confrontation requires the reduction of at least one of the confrontation’s Vitals to 0 ?

Classification

  • Classifications can be used to store information about the PC
